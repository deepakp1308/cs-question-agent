Main memory is the fast, directly-addressable storage (RAM) that the CPU reads program instructions and data from during execution.
In the Von Neumann model, main memory holds both program instructions and data in the same address space. The CPU fetches each instruction from main memory, decodes it, executes it, and may read or write data back to main memory. Main memory is usually RAM: it is fast, random-access, and volatile (contents are lost when power is removed).
  1. Main memory is organised as numbered storage locations called addresses.
  2. The CPU places the next instruction's address on the address bus.
  3. The instruction is copied from main memory into the CPU via the data bus.
  4. The CPU decodes and executes it, reading or writing data in main memory as needed.
Example: When you open a Python program, the operating system loads the program from the hard disk into main memory, and the CPU runs it from there.
Common mistake: Confusing main memory with secondary storage. Main memory (RAM) is fast and volatile; secondary storage (HDD/SSD) is slower and keeps data when power is off.
